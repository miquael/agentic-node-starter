9 - AI & automation security risks

AI systems, agents, and automation introduce new categories of security risk that must be explicitly addressed.

prompt injection
Malicious inputs can manipulate AI behavior. Systems must validate inputs and constrain outputs.

agent overreach
Agents with excessive permissions can perform unintended actions. Capabilities must be tightly scoped.

data leakage
Sensitive data may be exposed through AI interactions. Systems must redact and control inputs carefully.

tool chaining risks
Agents calling external systems can propagate vulnerabilities. All interactions must be validated.

automation abuse
Workflows and webhooks can be exploited. All triggers must be authenticated and rate-limited.
